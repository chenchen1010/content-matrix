# 如何参与改进内容矩阵框架

欢迎你来改进这个框架！以下是参与方式，**不需要任何编程知识**。

## 你能贡献什么

**欢迎提交：**
- 对工作流程（SKILL.md）的优化——比如发现某个步骤调整后效果更好
- 新增的功能模块——比如支持了一个新平台
- 提示词模板的改进——比如优化了小红书标题公式
- 脚本工具的改进——比如发布流程的优化

**不需要提交：**
- 你的业务数据（output/ 目录里的内容）
- 你的账号配置（.env、cookies.json）
- 仅适用于你个人业务的定制

## 怎么提交改进

### 方式一：让 Claude Code 帮你（推荐）

直接跟 Claude Code 说：

> "我觉得刚才的改动挺好的，帮我反馈给主框架"

Claude Code 会自动：
1. 分析你改了什么、为什么改、效果如何
2. **从你的改动中提炼出通用规则**——比如"标题加数字能提升点击率"，而不是"我的美妆品牌加数字后涨了 30%"
3. **给你过目**——展示提炼后的规则，你确认后才提交
4. 用中文写好说明，提交给主框架维护者审核

**提交的是提炼后的规则，不是你的代码改动。** 规则天然不包含你的业务信息——没有品牌名、没有客户数据、没有价格。整个过程你有两次确认机会（提炼后一次、提交前一次），任何时候觉得不对都可以叫停。

### 方式二：直接说明你的想法

如果你有改进建议但还没动手改，也可以在 GitHub 上提一个 Issue：

1. 打开 https://github.com/chenchen1010/content-matrix/issues
2. 点「New Issue」
3. 用中文描述你的建议：改了什么、为什么、效果如何

## 怎么获取框架更新

每次你打开项目使用 Claude Code 时，系统会自动检查是否有新版本。
如果有，你会收到中文提示，选择"帮我更新"就行。

你也可以随时说：

> "帮我检查一下框架有没有更新"

## 关于隐私保护

**回馈的是规则，不是你的代码。** 整个流程的设计就是从你的改动中提炼抽象的方法论：

- Claude Code 会分析你的改动，然后提炼出通用规则（比如"标题加数字能提升点击率"）
- **提炼的过程本身就是脱敏**——抽象规则天然不包含你的品牌名、客户数据、价格等任何业务信息
- 你有 **两次审核机会**：提炼结果看一次，正式提交前再看一次
- 任何时候你都可以说"算了不提交了"，不会有任何内容被公开
- 如果某个改动纯粹是你的个人业务定制，无法提炼为通用规则，Claude Code 会自动跳过

## 提交时的注意事项

- **写清楚为什么改**——比如"测试后发现标题加数字点击率提高"（不需要写具体数字，Claude Code 会帮你脱敏）
- **写清楚在什么场景验证的**——平台类型、大致效果即可
- 维护者会审核你的提交，可能会提出修改建议
- 审核通过后，所有使用这个框架的人都能用上你的改进
