# 内容生成引擎

从素材库中读取业务素材，结合选题方向，生成适配各平台的矩阵内容。

> **输出路径规则**：所有产出写入当前项目的 Obsidian vault。
> 目录结构见 `content-matrix/schemas/vault-structure.md`。

## 何时使用

当用户说"帮我生成内容"、"根据素材写笔记"、"生成矩阵内容"、"批量产出"时使用此 skill。

## 输入

使用前需要准备好两样东西：
1. **素材**：业务故事、客户案例、行业知识（来自素材库 skill 或用户直接提供）
2. **选题方向**：要写什么主题（来自选题 skill 或用户指定）

## 前置：加载知识底座

在进入工作流程前，先读取营销心理学底座：

```bash
cat "$(git rev-parse --show-toplevel)/content-matrix/skills/knowledge-base/marketing-psychology-foundation.md"
```

读取后，在 Step 4 生成内容时，根据目标和用户阶段，从底座的「附录 A 速查矩阵」中选取 2-3 个最适用的心理机制，融入内容设计。

## 工作流程

### Step 1: 确认输入

向用户确认以下信息：

```
需要你提供：
1. 素材来源：
   - 直接粘贴素材文本
   - 从 Obsidian 知识库自动检索（需指定关键词或选题方向）
   - 指定本地文件路径（如 ~/materials/客户案例.md）
   - 从飞书多维表格读取（需提供表格信息）
2. 选题方向：你要围绕什么主题生成内容？
3. 目标平台：小红书 / 抖音 / 公众号 / 视频号（可多选）
4. 生成数量：需要几个版本？（矩阵需要多个不同角度的版本）
5. 目标用户阶段（A层）：A1 / A2 / A3 / A4 / A5
   - 优先从选题报告中读取（素材库/5-选题报告/ 或 素材库/7-选题库/）
   - 若无选题报告，询问用户；若用户不确定，默认 A2
```

### Step 2: 读取素材

根据来源读取素材：

**方式一：用户直接提供**
用户粘贴文本，直接使用。

**方式二：从 Obsidian 知识库自动检索**

根据用户指定的选题方向，自动从 Obsidian 素材库中检索最相关的素材：

```bash
VAULT_PATH=$(obsidian-cli print-default --path-only)
MATERIAL_ROOT="$VAULT_PATH/素材库"

# 按选题关键词搜索相关素材
grep -r -l "{关键词}" "$MATERIAL_ROOT/" --include="*.md" 2>/dev/null

# 找出未使用或低频使用的素材（优先推荐）
grep -r -l "used_count: 0" "$MATERIAL_ROOT/" --include="*.md" 2>/dev/null
```

搜索到文件后，读取内容：
```bash
cat "$MATERIAL_ROOT/1-客户案例/教育/{搜索到的文件名}.md"
```

AI 对搜索结果进行相关性排序：
```
从以下素材中，选出与选题"{选题方向}"最相关的 1-3 条。
优先推荐 used_count 较低的素材。
给出推荐理由。
```

**方式三：本地文件**
```bash
cat {用户指定的文件路径}
```

**方式四：飞书多维表格**
```bash
CONFIG_FILE=~/.content-organizer/config.json

APP_ID=$(jq -r '.app_id' "$CONFIG_FILE")
APP_SECRET=$(jq -r '.app_secret' "$CONFIG_FILE")
TOKEN_RESP=$(curl -s -X POST "https://open.feishu.cn/open-apis/auth/v3/tenant_access_token/internal" \
  -H "Content-Type: application/json" \
  -d "{\"app_id\":\"$APP_ID\",\"app_secret\":\"$APP_SECRET\"}")
TOKEN=$(echo "$TOKEN_RESP" | jq -r '.tenant_access_token')

APP_TOKEN=$(jq -r '.app_token' "$CONFIG_FILE")
TABLE_ID=$(jq -r '.table_id' "$CONFIG_FILE")
curl -s "https://open.feishu.cn/open-apis/bitable/v1/apps/$APP_TOKEN/tables/$TABLE_ID/records?page_size=20" \
  -H "Authorization: Bearer $TOKEN" | jq '.data.items'
```

### Step 2.5: 卖点→买点三级转化

读取素材后，在生成内容前，先完成卖点到买点的转化。

用以下 prompt 处理素材：

```
你是一个消费心理专家。根据以下业务素材，完成卖点到买点的三级转化。

## 素材
{Step 2 获取的素材文本}

## 转化步骤
第一层 — 客观事实（卖点）：
  列出素材中已明确提到的产品/服务属性、参数、特点。
  【禁止】添加素材中没有的属性或数据。

第二层 — 客观感受：
  将每条客观事实转化为用户直接体验到的感受。
  格式："你会感到 ___" / "你不需要再担心 ___"
  【禁止】夸大或承诺效果，只描述感受。

第三层 — 主观想象（买点）：
  将客观感受转化为用户获得后的具体场景和情绪。
  格式："想象一下 ___" / "那一刻你会 ___"
  【禁止】发明不在素材中的场景，【禁止】加入效果数据或百分比。

## 输出格式
| 客观事实（卖点）| 客观感受 | 主观想象（买点）|
|---------------|---------|---------------|
| ...           | ...     | ...           |

买点摘要（用于 Step 4 的生成 prompt）：
{2-3 句话，提炼最有力的买点，用于内容生成}
```

将买点摘要保存为变量，在 Step 4 中使用。

### Step 3: 读取平台模板

根据目标平台，读取对应的提示词模板：
```bash
# 模板路径（相对于此 skill 目录）
cat templates/xiaohongshu.md   # 小红书
cat templates/douyin.md        # 抖音
cat templates/wechat-article.md # 公众号
cat templates/wechat-video.md  # 视频号
```

### Step 4: 生成内容

将素材 + 选题 + 平台模板 + A层口吻 + 买点组合，按以下指令生成内容：

```
你是一个资深的内容操盘手，擅长将业务素材转化为各平台的高效获客内容。

## 你的任务
根据以下素材和选题，按照平台模板的要求生成内容。

## 素材
{从 Step 2 获取的素材文本}

## 买点（用这个说服用户，不用卖点）
{从 Step 2.5 提炼的买点摘要}

## 选题方向
{用户指定的主题}

## 目标用户阶段：{A层}
{根据 A层 选择对应的口吻指引：}

A1 / A2 → 认知启蒙口吻
  - 不预设用户了解这个品类
  - 从用户已有的痛点/生活场景切入，引发共鸣
  - 语气：像朋友分享发现，不像在卖东西

A3 → 对比/测评口吻
  - 帮用户做选择，展示我们和其他选项的差异
  - 提供具体的判断维度或对比数据
  - 语气：专业、客观、有依据

A4 → 避坑/打消顾虑口吻
  - 直接点出用户最怕的那个疑虑
  - 用真实案例或具体细节消除顾虑
  - 语气：坦诚、细节多、有说服力

A5 → 用户证言/直接 CTA 口吻
  - 强化信任感（真实用户反馈、前后对比）
  - 有明确的行动号召
  - 语气：真实、直接、不拖泥带水

## 平台模板要求
{从 Step 3 读取的模板内容}

## 生成要求
1. 严格遵循模板中的结构、语气和字数要求
2. 融入素材中的真实细节（时间、地点、人物、数据）
3. 用买点（第三层）驱动用户情绪，而非罗列产品参数
4. 不要编造不存在的事实，不要添加素材中没有的属性
5. 输出格式清晰，可直接复制使用
```

### Step 5: 矩阵变体（如需多版本）

如果用户需要多个版本，按照模板中的"矩阵变体策略"生成不同角度的版本：

```
现在需要基于同一素材生成第 {N} 个变体版本。

变体维度（从模板中选择一个与之前不同的）：
- 视角切换：{具体切换方式}
- 表达风格：{具体风格}

要求：
1. 核心信息与第一版一致
2. 表达方式、结构、开头必须明显不同
3. 不能有大段重复的句子
4. 平台去重检测通过率要高
```

### Step 6: 合规自检

对每条生成的内容执行强制合规循环（最多 2 轮）：

**第 1 轮检查：**
```
请对以下内容进行合规自检：

{生成的内容}

检查项：
□ 是否包含广告法禁用词（最、第一、绝对、100% 等）
□ 是否有虚假宣传或承诺效果的表述
□ 是否有贬低竞品的内容
□ 是否涉及敏感话题
□ 是否有未脱敏的客户信息
□ 语气是否过度营销

如有问题，输出 "⚠️" + 具体问题说明。
如无问题，输出 "✅ 合规检查通过"。
```

**若第 1 轮结果含 ⚠️，强制执行重写：**
```
以下内容存在合规问题：
{第 1 轮检查结果}

请针对每个问题重写对应部分。要求：
- 只修改有问题的句子，保持其余内容不变
- 修改后语义和说服力不能大幅下降
- 输出完整重写版本
```

**第 2 轮检查：** 对重写版本重新执行同一检查。
- 若通过 → 输出 "✅ 合规检查通过（重写后通过）"
- 若仍含 ⚠️ → 标注 "⚠️ 仍有问题，需人工修改：{具体说明}"，不再重写

### Step 7: 输出

将最终内容按以下格式输出：

```
=== [平台名] 版本 [N] ===

【标题】
{标题}

【正文】
{正文内容}

【标签】
{标签列表}

【合规状态】✅ / ⚠️ {问题说明}

===========================
```

### Step 8: 写入 Obsidian 知识库

生成的每条内容自动写入当前项目 vault：

```bash
VAULT_PATH=$(obsidian-cli print-default --path-only)
PLATFORM="小红书"  # 或抖音/公众号/视频号
DATE=$(date +%Y-%m-%d)
VARIANT="性价比震撼型"  # 变体标签

OUTPUT_DIR="$VAULT_PATH/素材库/6-内容产出/$PLATFORM"
mkdir -p "$OUTPUT_DIR"

cat > "$OUTPUT_DIR/$DATE-$PLATFORM-$VARIANT.md" << 'EOF'
---
type: 内容产出
platform: {平台}
date: {日期}
topic: {选题}
variant: {变体标签}
source_material: "[[素材笔记名]]"
source_report: "[[选题报告笔记名]]"
status: 待发布
tags:
  - 内容产出
  - {平台}
---

# {标题}

## 正文
{正文内容}

## 标签
{标签列表}

## 合规状态
{合规检查结果}

## 变体信息
- 视角：{视角}
- 风格：{风格}
EOF
```

同时更新素材笔记的 `used_count` 和 `platforms_used`。

选题报告也写入：
```bash
REPORT_DIR="$VAULT_PATH/素材库/5-选题报告"
mkdir -p "$REPORT_DIR"
# 将 Step 3 的选题分析写入 $REPORT_DIR/$DATE-{关键词}-{平台}选题报告.md
```

## 快捷用法

用户可以用简短指令触发：
- "用这段素材生成小红书笔记" → 跳到 Step 4
- "从知识库找素材写一条抖音" → Step 2（Obsidian 检索）→ Step 4
- "再给我3个不同角度的版本" → 跳到 Step 5
- "检查一下这条内容合不合规" → 跳到 Step 6
