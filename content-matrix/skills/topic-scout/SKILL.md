# 选题侦察

从小红书搜索同类爆款内容，通过 4 轮迭代关键词扩展，输出 A1-A5 用户决策阶段标注的完整内容情报：客户画像表、长尾关键词库、对标爆款笔记库、选题库。

## 何时使用

当用户说"帮我找选题"、"搜一下爆款"、"看看最近什么内容火"、"找灵感"时使用此 skill。

## 前置条件

小红书 MCP 已安装并运行：
```bash
mcporter list 2>/dev/null | grep -i xiaohongshu || echo "❌ 小红书 MCP 未启动，请参考 skills/tools/search/xiaohongshu/README.md"
```

如显示 `offline`，不要把终端命令复制给用户。应：
1. 运行 **platform-login** skill，由助手执行登录脚本，或提示用户 **Cmd+Shift+P → Tasks: Run Task →「内容矩阵 · 小红书登录」**
2. 登录完成后由助手启动 `xiaohongshu-mcp`（任务：**「内容矩阵 · 启动小红书 MCP」**）

## A层分类表（A0-A5）

每条内容和每个关键词，按信号词判断所属 A层：

| A层 | 含义 | 信号词（关键词或标题含以下词） |
|-----|------|-------------------------------|
| **A0** | 完全不相关 | 不符合任何 A1-A5 信号 → **过滤，不存档** |
| **A1** | 认知启蒙 | 泛词，不含品类认知（如"家里脏了怎么办"、"家政是什么"、行业大词无修饰） |
| **A2** | 研究阶段 | 好不好、怎么样、有没有必要、值不值得、需不需要、效果、入门、了解、知道 |
| **A3** | 对比选择 | 怎么选、对比、哪家好、推荐、哪个好、测评、排行、选哪个 |
| **A4** | 决策顾虑 | 避坑、踩坑、靠不靠谱、投诉、差评、黑名单、坑、骗、后悔、注意 |
| **A5** | 准备成交 | 品牌词 + 价格、预约、电话、多少钱、怎么联系、地址、报名 |

> **规则**：差评区/评论区中提取的关键词**始终标注为 A4**，无论词面如何。
>
> **默认值**：Round 1-2 中无法判断的词默认 A2；Round 3-4 中无法判断的词默认 A3。

## 工作流程

### Step 1: 业务背景对齐 + 项目知识库初始化

> **核心原则**：调研必须服务于用户的具体业务目标。不了解业务身份就做调研，产出的选题方向很可能与用户需求错位。

#### 1a. 检查项目知识库

首先检查项目目录下是否已有 `业务知识库.md`：
- **已有** → 读取，用已有信息跳过重复提问，只补充缺失项
- **没有** → 创建，通过下方问题引导用户填充

项目知识库路径：`{项目名}/业务知识库.md`

> 项目知识库是这个项目的"业务大脑"，来源包括：用户主动提供、工作流引导用户回答、调研过程中积累的市场认知、用户之前沉淀的资料。随时更新。

#### 1b. 引导用户提供业务信息

向用户提问（**必须在 Step 2 之前完成**）。不要一次性抛出所有问题——根据用户回答的深度逐步追问。先问身份，再根据身份追问细节。

**第一轮（必问）：**
```
1. 你的业务是什么？目标城市/地区？
2. 你在这个市场里是什么角色？
   - 官方/平台方（如：文化馆、政府项目）
   - 第三方商业机构（如：自营夜校、培训机构）
   - 个人老师/自由职业者
   - 信息聚合/导购平台
   - 纯内容号（不卖课，靠流量变现）
3. 你发笔记的核心目的？（获客引流 / 品牌认知 / 直接成交）
```

**第二轮（根据身份追问）：**
```
针对第三方商业机构：
  - 品牌名？目前开设了哪些课程/服务？价格区间？
  - 覆盖哪些区域？
  - 与同类竞品、与官方/免费替代品的差异化优势？
  - 现有小红书账号？发过什么内容？哪些效果好/不好？
  - 目前获客转化路径？（小红书→私信→微信？）

针对个人老师：
  - 教什么？在哪里教？价格？
  - 有没有学员案例/作品？

针对官方/平台：
  - 课程更新频率？报名机制？
  - 内容目的是告知还是招生？
```

**第三轮（如果用户提供了资料）：**
用户可能提供：官网链接、产品手册、之前的笔记、竞品截图等。将这些信息整理进业务知识库。

#### 1c. 写入项目知识库

将收集到的信息写入 `{项目名}/业务知识库.md`，包含：
- 业务身份
- 市场结构（调研中持续补充）
- 竞品信息（调研中持续补充）
- 差异化优势
- 待补充信息清单

> **为什么要做这些**：同一个关键词"杭州夜校"，官方文化馆和第三方夜校的内容策略完全不同——官方号发"报名攻略"就够了（用户自来），第三方需要建立信任、差异化、制造转化路径。不同身份关注的竞品、参考的爆款、适配的内容模板完全不一样。

#### 1d. 确认搜索关键词

基于业务身份确定初始关键词（1-3 个）。平台固定为**小红书**。

### Step 2: 4 轮迭代搜索

#### 内部数据结构

```
关键词池：       本轮待搜索的关键词列表
已搜关键词集：   所有已完成搜索的关键词（去重用）
长尾关键词库：   {词: {a_layer, search_round, source}}
爆款笔记暂存：   {a_layer: [笔记列表]}（每层最多 20 条，信息饱和后停止）
```

#### 搜索执行器（3 级降级）

每搜一个关键词，按顺序尝试，成功即停：

```
Level 1: xiaohongshu-mcp（优先）
  mcporter call 'xiaohongshu.search_feeds(keyword: "{关键词}")'

Level 2: Jina Reader
  curl -s "https://r.jina.ai/https://www.xiaohongshu.com/search_result?keyword={URL编码关键词}" \
    -H "Accept: text/plain" -H "X-Return-Format: text"
  （结果 < 50 字符或含 SecurityCompromiseError → 降级）

Level 3: cursor-ide-browser
  browser_navigate → browser_wait_for（最多 10 秒）→ browser_snapshot → 提取标题和点赞
  （页面要求登录 → 提示用户在浏览器中登录小红书）
```

#### 轮次循环

```
初始化：
  关键词池 = [用户提供的初始关键词]
  已搜关键词集 = {}

Round 1 → Round 4（逐轮执行）：

  1. 对关键词池中每个词执行搜索
  2. 对每条搜索结果：
     a. 按 A层分类表 判断 A层
     b. 若 A层 = A0 → 跳过
     c. 若 A层 = A1-A5 → 暂存到 爆款笔记暂存[a_layer]（信息饱和则跳过，见 Step 3）
     d. 从标题/正文/标签提取新词 → 候选词池
     e. 挖掘差评区/评论区（见下方）→ 提取 A4 词
  3. 新词 去重（剔除 已搜关键词集 中已有的词）→ 本轮新增词
  4. 将本轮搜索词移入 已搜关键词集
  5. 将本轮新增词加入下一轮关键词池

  停止条件（任一满足即停止迭代）：
  - 本轮新增词数量 / 本轮候选词总数 < 20%（即重叠率 ≥ 80%）
  - 已完成 4 轮
```

#### 素人号/同类竞品扫描（每轮顺带执行）

> **核心洞察**：低互动但持续更新的模板化内容 ≠ 无价值。如果一个素人号反复发同一种模板，说明这套模板在持续带来转化。**只有爆款才值得重复。**

搜索过程中，除了收集高赞爆款，还要**识别和记录同类竞品账号**：

```
识别信号（满足 2 个以上即标记为「竞品素人号」）：
  1. 昵称含品牌名/品类词/地域词（如"杭州成人书法"、"知行夜校方校长"）
  2. 多条笔记互动低（0-50 赞）但持续在发
  3. 同一品牌有多个账号（矩阵号：主号+老师号+学员号）
  4. 标题高度模板化（如"XX夜校——{区域}{课程}✔"）
  5. 两条几乎一样的内容（关键词排列组合变体）

对每个竞品素人号记录：
  - 账号昵称 / 账号ID
  - 竞品类型：与用户同类 / 上游 / 下游 / 替代品
  - 发帖模板（提取标题公式）
  - 发帖频率（搜索中出现的条数）
  - 互动水平（平均赞数）
  - 业务模式推断（获客→私信？挂链接？评论区引流？）
```

**汇入输出**：竞品素人号数据汇入 Step 4 的选题报告「竞品图谱」章节。

#### 差评区挖掘（每条笔记顺带执行）

```bash
# 拉取评论（前 20 条）
mcporter call 'xiaohongshu.get_note_comments(note_id: "{id}")'
```

提取含以下词的评论：`避坑、踩坑、不靠谱、投诉、差评、后悔、骗、黑心、注意`

- 将提取的关键词标注 **A4**，加入长尾关键词库（`source: "comment"`）
- 涉及笔记记录为差评来源，列入 A4 爆款参考暂存

#### 关键词 A层标注规则

提取新词时按来源自动标注（若含信号词则用信号词判断，否则用默认值）：

| 来源 | 默认 A层 |
|------|---------|
| Round 1 提取 | A2 |
| Round 2 提取 | A2 |
| Round 3-4 提取 | A3 |
| 差评区提取 | A4（强制，不覆盖） |

所有词加入**长尾关键词库**，记录：`{词, a_layer, search_round, source: "content|comment"}`

### Step 3: 信息饱和检验 + 爆款笔记存档

对每条笔记在写入爆款参考前执行饱和检验：

#### 饱和检验规则

```
每个 A层 维护一个「已覆盖主题列表」（1-2 句话描述每个主题）。

存入前：
  1. 提取该笔记的核心主题（1-2 句话）
  2. 与已覆盖主题列表比较重叠度：
     - 重叠度 ≥ 80% → 跳过（不存，主题已覆盖）
     - 全新角度 → 存入，更新已覆盖主题列表
  3. 单层上限：20 条（达到后该层停止新增）
```

#### 获取笔记详情

对通过饱和检验的笔记，调用详情接口：
```bash
mcporter call 'xiaohongshu.get_feed_detail(feed_id: "{id}", xsec_token: "{xsecToken}")'
```

提取：标题、正文全文、标签、图片 URL、视频 URL 和时长、点赞/收藏/评论/分享、作者、精选评论（前 5 条）

#### 存为爆款参考笔记

路径：`素材库/4-爆款参考/小红书/{标题}.md`

**原文链接格式**（必须带 xsec_token）：
